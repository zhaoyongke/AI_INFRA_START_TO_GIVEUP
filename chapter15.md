## 15.2 如何与算法工程师有效沟通

### 理解彼此的关注点

```
┌─────────────────────────────────────────────────────────────┐
│               Infra vs 算法 关注点差异                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  算法工程师关注：                                             │
│  ─────────────────────────────────────────────────────────  │
│  ✓ 模型精度、效果                                          │
│  ✓ 新模型架构、新技术                                      │
│  ✓ 训练数据处理                                            │
│  ✓ 论文复现                                                │
│                                                             │
│  常见问题：                                                   │
│  - "为什么模型效果变差了？"                                 │
│  - "这个新架构你们支持吗？"                                │
│  - "训练速度能更快吗？"                                    │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  Infra 工程师关注：                                           │
│  ─────────────────────────────────────────────────────────  │
│  ✓ 系统稳定性、可用性                                       │
│  ✓ 性能、资源利用率                                        │
│  ✓ 成本优化                                                │
│  ✓ 可扩展性、可维护性                                      │
│                                                             │
│  常见问题：                                                   │
│  - "这样做会不会影响稳定性？"                               │
│  - "资源消耗是多少？"                                      │
│  - "有没有更节省成本的方案？"                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 沟通技巧

```
┌─────────────────────────────────────────────────────────────┐
│                    沟通技巧总结                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 说对方的语言                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  ❌ 不要说："KV Cache 碎片化导致 OOM"                       │
│  ✓ 要说："生成的文本太长，内存不够用了"                     │
│                                                             │
│  ❌ 不要说："P2P 通信失败"                                  │
│  ✓ 要说："多卡训练时显卡之间通信有问题"                     │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  2. 提供数据而非观点                                         │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  ❌ "这个改动会影响性能"                                     │
│  ✓ "测试显示延迟增加 15%，从 200ms 到 230ms"               │
│                                                             │
│  ❌ "成本很高"                                              │
│  ✓ "每个请求成本增加 20%，每 1k tokens 从 $0.0001 到 $0.00012"│
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  3. 提供选项供选择                                           │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  ❌ "这个实现不了"                                          │
│  ✓ "有几个方案：                                           │
│     A. 快速方案（1天），精度损失 2%                         │
│     B. 完整方案（1周），保持精度                            │
│     C. 折中方案（3天），精度损失 0.5%                       │
│     你倾向哪个？"                                           │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  4. 先理解再反驳                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  模板：                                                      │
│  "我理解你希望 X，因为 Y 对吧？                             │
│   我的担忧是 Z，因为历史教训是 W。                          │
│   我们有没有办法同时满足 X 和避免 Z？"                      │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  5. 共同目标                                                 │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  提醒大家：                                                   │
│  - 目标是产品成功，不是技术完美                             │
│  - 需要在精度、性能、成本之间权衡                           │
│  - 一起寻找最优解                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 本章小结

1. **成长有路径**：从初级到高级，每个阶段有不同的能力要求和发展建议。

2. **沟通很重要**：理解算法同事的关注点，用对方的语言沟通，提供数据而非观点。

3. **开源有价值**：参与开源能提升能力、建立影响力，从小的贡献开始。

4. **放弃幻想**：接受 AI Infra 的复杂性，保持谦逊和好奇心，持续学习。
